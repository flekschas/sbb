About

Semantic Body Browser - a tool for graphically exploring an organism's body. The tool has been developed as part of the CellFinder project. An integrated version can be found at cellfinder.org/browse. The source code is provided on Github.

Abstract

Advancing technologies generate large amounts of molecular and phenotypic data on cells, tissues and organisms, leading to an ever-growing detail and complexity while information retrieval and analysis becomes increasingly time-consuming. Most data resources provide text-based information retrieval only, meaning that the user searches for key words of a desired target. While this is sufficient when the target is known by name, it often fails or is time-consuming when the exact target name is unknown. Similarly, data interpretation based on large tables of numbers is suitable for computational analysis, but requires thorough background knowledge and time to be analysed manually. Often, quick access to information is desirable to evaluate the benefits of further research.

The Semantic Body Browser (SBB) is a web application for intuitively exploring the body of an organism from the organ to the sub-cellular level and visualising expression profiles by means of semantically annotated illustrations. It is used to comprehend biological and medical data related to the different body structures while relying on the strong pattern recognition capabilities of human users. A simple mouse click highlights the target and makes it possible to retrieve information or browse further along one of the four dimensions: resolution, development, species or gender.

All biological entities are annotated using CELDA1 ontology.

  1. 1 CELDA - an ontology for the comprehensive representation of cells in complex systems. BMC Bioinformatics (2013), 14, 228. (Seltmann et al.).
  2. 2 CellFinder: a cell data repository. Nucleic Acid Res. (2013), 42, D950-D958. (Stachelscheid et al.).

Browsing Tree

The Semantic Body Browser currently incorporates 22 views for homo sapiens and 21 views for mus musculus. These views branch into 15 different levels of resolution and 6 developmental stages. The adult human body gross view does furthermore distinguish genders.

Heat map of RNA abundance

Color Range

Min Max 50%

The colors are relative to the accumulative RNA expression values of the selection of genes. The minimum displayed by a dark blue and the maximum by a bright yellow.

The heat map modus lets you colour the human adult body gross view according to the spatially resolved RNA expression value.

The organs and anatomical parts for which data is available will be coloured relatively to the accumulative RNA expression value of the user-defined selection of genes. If multiple genes have been selected a stacked bar indicates the proportion of each gene. Sub-selection facilitates the comparison among the set of genes and helps to quickly identify the impact of one genes on the accumulative RNA expression value.

Currently transcriptome sequencing data is provided by the RNA-Seq Atlas1 and the Human BodyMap 2.02. As shown by Li et al.3 and Wagner et al.4 RPKM may be biased across different samples, therefore we provide TPM.

  1. 1 RNA-Seq Atlas - A reference database for gene expression profiling in normal tissue by next generation sequencing. Bioinformatics (2012), 28, 1184-1185. (Krupp, M. et al.).
  2. 2 (Ensembl Release 74)
  3. 3 RNA-Seq gene expression estimation with read mapping uncertainty. Bioinformatics (2010), 26, 493-500. (Li, B. et al.)
  4. 4 Measurement of mRNA abundance using RNA-seq data: RPKM measure is inconsistent among samples. Theory Biosci (2012), 131, 281-285. (Wagner, GP. et al.).

Implementation

The SBB is a JavaScript-based web application which is implemented using the AngularJS framework provided by Google. For cross-browser drawing and manipulation of SVG we make use of Raphaël. General DOM handling is realised using jQuery. Additionally we use the following smaller JavaScript libraries:

  • Mousewheel: Utilising the mousewheel.
  • Hammer.js: Touch event library
  • Blob.js: Cross-browser implmentation of the blob interface. (Used by FileSaver.)
  • FileSaver.js: Cross-browser file saving interface.

Templates are written in HTML5 and CSS 3 including the great normalize.css for rendering elements more consistently across multiple browsers.

Our server-side php-based RESTful API is implemented using the Slim framework. Our data is stored using a MySQL server.

The source code is available on GitHub.

Compatability

The Semantic Body Browser is compatible with devices having a screen resolution of at least 600×4001 px and either one of the following browser.

  • 1

    The dimension refers to the device display dimension and not the actual physical pixel. E.g. a device which displays 1440×960 px at a density of 3 does have a device display dimension of only 480×320 px and is thus not supported.
